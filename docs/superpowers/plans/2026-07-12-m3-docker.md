# Umbra M3 — Docker Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** `docker` and `docker compose` on the macOS host work unchanged against a dedicated Umbra-managed dockerd VM — `umbra docker start` boots it, bridges its socket to `~/.umbra/run/docker.sock`, and registers/activates docker context `umbra`.

**Architecture:** A reserved ordinary `registry.Machine` named `docker` (distinguished by a `Role` field + a docker-flavored cloud-init profile that installs dockerd via get.docker.com, exposes it on `tcp://0.0.0.0:2375`, and firewalls that port to the gateway IP only). `umbrad` runs a host unix-socket listener (`internal/dockerbridge`) that pipes each connection to `dockerVMIP:2375` via the existing `netstack.Stack.DialContextTCP` — gvisor-tap-vsock's `Expose` is TCP-only and can't bind a host unix socket. An `umbra docker install|start|stop` command group layers the bridge + idempotent `docker context` registration on top of the normal `Manager.Start(ctx,"docker")`, keeping `vm.Manager` docker-agnostic.

**Tech Stack:** Go 1.25, the existing netstack/vm/cloudinit/registry packages, the host `docker` CLI (brew).

## Global Constraints

- Research cheat-sheet is authoritative: `docs/research/dockerd-in-vm.md`. Do not deviate from its verified findings (esp. §2 — gvisor `Expose` can't do host unix sockets; §7 pitfalls P13–P17).
- **Security (non-negotiable, research §2 "Security"):** dockerd's TCP 2375 API is unauthenticated and every VM shares the `192.168.127.0/24` L2 (including the untrusted `fwb-ci` runner). The docker cloud-init MUST include `iptables -A INPUT -p tcp --dport 2375 ! -s 192.168.127.1 -j DROP` so only the Umbra host (gateway IP) can reach it. A docker VM without this rule is a defect, not a feature.
- Reserved name `docker`: user `umbra create docker` / API create must reject it; only `umbra docker install` may create it.
- Bridge socket at `paths.Run()/docker.sock`, 0600, stale-file-removed before bind (P14).
- docker context `umbra` registration is idempotent (create-or-update), re-asserted `use` on every start (research §3, Colima behavior). `umbra docker uninstall`/`rm docker` must `docker context rm umbra` (P15).
- `umbra docker` commands require the host `docker` CLI — check `exec.LookPath("docker")` up front with a `brew install docker` hint.
- docker-readiness: after SSH-ready, poll `DialContextTCP(dockerIP:2375)` + Engine `GET /_ping` before reporting started (P13) — a new named stage, same bounded shape as `vm.WaitReady`.
- **Deferred to a later milestone (out of M3 scope):** docker-event-driven per-container `<container>.umbra.local` DNS + auto-published-port forwarding (research §7 P18). M3 delivers the VM+socket+context foundation; note the deferral in README.
- Build/test discipline identical to M1/M2: `//go:build darwin && arm64` where vz is touched; integration/E2E on this Mac only; `gofmt`+`go mod tidy`+`make lint`+`go test ./... -count=1 -race` before each commit; conventional commits + trailers; specific `git add`.
- Every new background goroutine (the bridge) is daemonCtx + WaitGroup shutdown-wired, like M2.

```go
package dockerbridge

import (
	"bufio"
	"context"
	"net"
	"os"
	"path/filepath"
	"testing"
	"time"
)

type tcpDialer struct{ addr string }

func (d tcpDialer) DialContextTCP(ctx context.Context, _ string) (net.Conn, error) {
	return (&net.Dialer{}).DialContext(ctx, "tcp", d.addr)
}

func TestBridgePipesToGuest(t *testing.T) {
	// echo server standing in for dockerd's TCP API
	echo, err := net.Listen("tcp", "127.0.0.1:0")
	if err != nil {
		t.Fatal(err)
	}
	defer echo.Close()
	go func() {
		for {
			c, err := echo.Accept()
			if err != nil {
				return
			}
			go func() { io.Copy(c, c); c.Close() }()
		}
	}()

	sock := filepath.Join("/tmp", "umbra-br-test.sock")
	os.Remove(sock)
	if err := os.WriteFile(sock, []byte("stale"), 0o600); err != nil { // stale file present
		t.Fatal(err)
	}
	b, err := Listen(tcpDialer{echo.Addr().String()}, sock, "192.168.127.10:2375")
	if err != nil {
		t.Fatalf("listen (stale removal): %v", err)
	}
	defer b.Close()
	ctx, cancel := context.WithCancel(context.Background())
	defer cancel()
	go b.Serve(ctx)

	c, err := net.DialTimeout("unix", sock, 2*time.Second)
	if err != nil {
		t.Fatal(err)
	}
	defer c.Close()
	c.Write([]byte("ping\n"))
	line, _ := bufio.NewReader(c).ReadString('\n')
	if line != "ping\n" {
		t.Fatalf("echoed %q", line)
	}
}
```
